# 架构文档

## 核心理念

- 事件即标签，时间块是原子单位
- 数据即事实，无意义赋予
- 呈现而非分析
- 过犹不及，不过度设计

---

## 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    React Frontend                     │
│  ┌─────────────────────────────────────────────────┐ │
│  │              滑动层 (Swiper)                     │ │
│  │  ┌─────────────────────────────────────────────┐│ │
│  │  │         垂直维度 (功能页面)                   ││ │
│  │  │  History ←→ Timer ←→ Settings ←→ History   ││ │
│  │  │                   ↑                         ││ │
│  │  │         水平维度 (预制计时器)                 ││ │
│  │  │  Timer35 ←→ Timer90 ←→ TimerCountup ←→ ... ││ │
│  │  └─────────────────────────────────────────────┘│ │
│  └─────────────────────────────────────────────────┘ │
│                         ↓                            │
│                    Zustand Store                     │
└───────────────────────────┬─────────────────────────┘
                            │ tauri-plugin-sql
                            ↓
┌─────────────────────────────────────────────────────┐
│                      SQLite                         │
└─────────────────────────────────────────────────────┘
```

**两维正交，独立循环：**
- 垂直维度：功能页面（History / Timer / Settings）
- 水平维度：预制计时器（同等地位，可增删）
- 两维互不交叉，各自循环

---

## 数据库模型

### events（事件标签）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT (UUID) | PK |
| name | TEXT | 事件名称 |

### timer_presets（预制计时器）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT (UUID) | PK |
| name | TEXT | 显示名称（可选） |
| timer_type | TEXT | countdown / countup |
| default_duration | INTEGER | 默认时长（秒），倒计时用 |
| default_event_id | TEXT | FK → events.id（可选） |
| order_index | INTEGER | 水平环顺序 |

### sessions（计时记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT (UUID) | PK |
| preset_id | TEXT | FK → timer_presets.id（可选） |
| event_id | TEXT | FK → events.id（可空，计时结束后填写） |
| start_time | INTEGER | Unix ms |
| end_time | INTEGER | Unix ms |
| duration | INTEGER | 实际时长（秒） |
| pause_count | INTEGER | 暂停次数 |
| pause_duration | INTEGER | 累计暂停时长（秒） |
| timer_type | TEXT | countdown / countup |
| is_completed | INTEGER | 是否正常结束（0/1） |

## 页面结构

### 垂直维度（功能页面）

| 页面 | 用途 |
|------|------|
| Timer | 计时器主页面 |
| History | 历史记录与可视化 |
| Settings | 全局设置 |

### 水平维度（预制计时器）

- 数量不固定，用户可增删
- 所有计时器地位等同
- 长按进入当前计时器的设置模式

---

## 交互逻辑

### 滑动

| 平台 | 操作 | 效果 |
|------|------|------|
| 移动端/触屏 | 左右滑动 | 切换预制计时器 |
| 移动端/触屏 | 上下滑动 | 切换功能页面 |
| 桌面端 | 鼠标移至边缘箭头 | 显示导航选项 |
| 桌面端 | 左键点击箭头 | 下一页/下一个计时器 |
| 桌面端 | 右键点击箭头 | 上一页/上一个计时器 |
| 桌面端 | 箭头处hover停留 | 唤出选单 |

### 计时器设置（长按进入）

- 切换类型：倒计时 ↔ 正向计时
- 设置时长：数字输入（仅倒计时）
- 绑定默认事件：选择已有事件或新建

### 计时结束

- 弹窗提示计时结束
- 填写事件名称（可选，可跳过）
- 数据存入sessions表

### 历史补填

- 在History页面可补填过往session的事件

## 技术实现要点

| 模块 | 方案 |
|------|------|
| 滑动框架 | react-swipeable 或 framer-motion |
| 热力图 | 开源库（如 react-github-contributions） |
| 状态管理 | Zustand |
| 数据库 | tauri-plugin-sql (SQLite) |
| IPC | Tauri invoke |
| 计时精度 | JS setInterval（不做极限要求） |

---

## Rust端职责

- 窗口管理（最小化、置顶、托盘）
- SQLite读写（通过 plugin）
- 文件导出（JSON/CSV）

---

## 不做的事情

- 不做任务层级/分组
- 不做打卡/成就/激励
- 不做自动分析/趋势解读
- 不做云端同步（当前）
- 不过度抽象，不过度设计
